# Ноды и импорт

## Добавление ноды

Для каждой ноды укажите:

- code — короткий уникальный код;
- name — понятное название;
- country — код страны;
- base_url — полный URL с портом;
- token — значение NODE_AGENT_TOKEN;
- enabled — участвует ли нода в выдаче доступа;
- is_local — находится ли AWG на сервере главной панели.

После сохранения нажмите «Проверить». Не начинайте массовую выдачу конфигураций, пока agent API не отвечает стабильно.

## Импорт существующей ноды

1. Добавьте и проверьте ноду.
2. Нажмите «Импорт».
3. Главная панель запросит /api/node-agent/export.
4. Пользователи объединятся по username.
5. Существующие устройства сохранят имена, адреса и ключи.

Перед импортом приведите username к единому виду. Значения Ivan, ivan и ivan  лучше исправить заранее, чтобы избежать неоднозначного объединения.

Если один username найден на нескольких странах, в главной панели создаётся один пользователь с доступом к нескольким нодам. Существующие peers не должны получать новые ключи без необходимости.

## Проверка после импорта

1. Откройте пользователя и проверьте «Доступные серверы».
2. Сверьте список устройств.
3. Скачайте конфигурацию каждой страны.
4. Убедитесь, что старые клиенты продолжают подключаться.
