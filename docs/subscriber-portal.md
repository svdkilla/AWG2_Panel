# Страница подписки и брендинг

Для каждого пользователя создаётся секретная ссылка:

```text
http://<host>:51351/u/<public_token>
```

Страница не требует логина. Доступ защищает сам токен в URL, поэтому не публикуйте ссылку открыто.

## Что видит клиент

- состояние и срок подписки;
- лимит устройств;
- использованный трафик;
- доступные страны;
- список устройств;
- QR-коды, .conf и vpn://.

Клиент может создать устройство, выбрать страну, переименовать или удалить устройство и получить конфигурацию.

## Параметры брендинга

ПолеНазначениеpublic_nameназвание в шапке порталаpublic_subtitleподпись под названиемsubscription_title_templateзаголовок карточки подпискиconfig_filename_prefixпрефикс файла .confconfig_description_templateназвание подключения в Amnezia

Доступные переменные шаблонов:

```text
{brand}
{username}
{device}
{address}
```

Пример:

```json
{
  "public_name": "MyVPN",
  "public_subtitle": "Доступ к VPN",
  "subscription_title_template": "Подписка {username}",
  "config_filename_prefix": "myvpn",
  "config_description_template": "{brand} · {device}"
}
```

Брендинг меняет клиентский портал и выдаваемые файлы, но не переименовывает административную панель XConnect.
